## Objective

Create comprehensive user-facing documentation that enables developers to set up, configure, and use the ServiceNow MCP Bridge. Update the README with setup instructions, configuration reference, supported tools listing, and authentication setup guide. Create a `.env.example` file documenting all configuration options. Provide usage examples for common workflows.

## Acceptance Criteria

- README.md is updated with:
  - Quick start guide (install, configure, run).
  - Prerequisites (Node.js >=20, ServiceNow instance with REST API access).
  - Installation instructions (clone, npm install, build).
  - Configuration section with all environment variables and their defaults.
  - Supported tools table listing all tools across all modules with descriptions.
  - Authentication setup guide for each strategy (Basic, OAuth, Token).
  - Usage examples showing how to configure the server in Claude Desktop, Cursor, and other MCP clients.
  - Troubleshooting section covering common issues (auth failures, missing permissions, rate limiting).
- `.env.example` file is created with all configuration options, organized by category, with inline comments explaining each option and its default value.
- CLAUDE.md is updated to reflect the completed project state (commands for build, test, run).
- Usage examples are practical and copy-pasteable.
- No credentials or real instance URLs appear in documentation.

## Technical Notes

- **README structure**: Follow a standard open-source project README pattern:
  1. Project title and badges (if applicable)
  2. One-paragraph description
  3. Features list
  4. Quick start
  5. Prerequisites
  6. Installation
  7. Configuration
  8. Authentication setup
  9. Supported tools (table format)
  10. Usage with MCP clients
  11. Development (build, test, lint)
  12. Troubleshooting
  13. Contributing
  14. License
- **.env.example format**:
  ```bash
  # ServiceNow Instance Configuration
  SERVICENOW_INSTANCE=your-instance-name  # Instance name (e.g., "mydev" for mydev.service-now.com)

  # Authentication
  SERVICENOW_AUTH_TYPE=basic  # Options: basic, oauth, token
  SERVICENOW_USERNAME=        # Required for basic auth
  SERVICENOW_PASSWORD=        # Required for basic auth
  # SERVICENOW_CLIENT_ID=     # Required for oauth
  # SERVICENOW_CLIENT_SECRET= # Required for oauth
  # SERVICENOW_TOKEN=         # Required for token auth

  # Module Configuration
  # MODULES_KNOWLEDGE_ENABLED=true
  # MODULES_KNOWLEDGE_ALLOW_WRITE=false
  # ... (all modules)
  ```
- **MCP client configuration examples**: Show the `mcpServers` JSON configuration for:
  - Claude Desktop (`claude_desktop_config.json`)
  - Cursor (`.cursor/mcp.json`)
  - Generic stdio MCP client
- **Supported tools table**: Generate from the implemented modules. Include tool name, module, description, and whether it requires write access.
- **Haiku-suitable**: Documentation tasks follow clear templates and patterns. The content is derived from existing code and specifications -- no architectural decisions required.
